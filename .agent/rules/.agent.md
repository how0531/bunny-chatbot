## 0. 核心原則 (Prime Directives)

1. **數據誠信鐵律**：所有數據必須來自真實數據源（MongoDB/ClickHouse/yfinance/Tavily），嚴禁編造
2. **零幻覺原則**：財報數字必須可追溯，不確定時標註「預估」或「暫無資料」
3. **合規優先**：任何建議必須附風險警語，禁止保證獲利承諾
4. **效能要求**：API 回應時間 < 5秒，UI 渲染無卡頓
5. **可測試性**：每個功能必須有明確的觸發指令與驗證標準

## 2. Agent 協作工作流程

### 標準個股分析流程

```
用戶輸入: "2330"
    ↓
[@Sophia] 意圖識別 → 判定為「個股深度分析」
    ↓
[@Sophia] 任務分發 → 同時啟動:
    ├─ [@Arthur] 計算 BunnyGood 分數 → 返回 {score: 8.5, metrics: {...}}
    ├─ [@Isabella] 搜尋最新新聞 → 返回 {summary: "法說會利多", sources: [...]}
    └─ [@Arthur] 查詢籌碼數據 → 返回 {flow: {foreign: +5000, trust: +1200}}
    ↓
[@Oscar] 合規檢查 → 驗證數據一致性 → 返回 {validation_passed: true}
    ↓
[@Sophia] 報告整合 → 生成最終 Markdown 回應
    ↓
[@Leo] UI 渲染 → 顯示卡片式報告 + 圖表
    ↓
[@Kevin] 監控 → 記錄回應時間、檢查錯誤日誌
    ↓
返回給用戶
```

### 條件篩選流程

```
用戶輸入: "找出本益比<15且外資連買3天的股票"
    ↓
[@Sophia] 意圖識別 → 判定為「複合條件篩選」
    ↓
[@Arthur] RuleEngine 解析 → 提取條件: {pe: <15, foreign_buy_days: >=3}
    ↓
[@Arthur] 執行查詢 → 從 MongoDB/ClickHouse 撈取符合條件的股票
    ↓
[@Arthur] 資料豐富化 → 並行獲取每檔股票的詳細數據 (ThreadPoolExecutor)
    ↓
[@Oscar] 雜訊過濾 → 剔除處置股、警示股
    ↓
[@Sophia] 結果格式化 → 生成表格式回應
    ↓
返回給用戶
```
